# 首页首屏“今日时间轴”重设计

- 日期：2026-07-14
- 状态：已确认
- 范围：首页首屏主体的左、右两块

## 1. 目标

让用户打开首页后，第一眼就能回答两个问题：

1. 现在是什么时间？
2. 当前最重要、最近发生的日子有哪些？

首屏从“宣传文案 + 单个圆盘倒计时”调整为“现在 + 重要时间”的信息型布局，同时继续沿用项目现有的 Fluffy 奶油色毛玻璃风格。

## 2. 范围与非目标

### 本次修改

- 重构 `hero-stage` 内的左侧当前时间区域。
- 重构 `hero-stage` 内的右侧置顶圆盘区域。
- 调整首屏数据选择与渲染逻辑，使右侧展示三条重要时间。
- 为新布局补充桌面端、平板端和手机端响应式样式。

### 明确不修改

- 顶部浮动栏及新增、同步、登出操作。
- 下方事件列表、分类筛选、拖拽排序和卡片操作。
- 新增或编辑事件弹窗。
- 登录、会话、API、D1 数据结构及持久化方式。
- 单一置顶约束、节假日只读规则和现有写操作防抖机制。

## 3. 信息架构

首屏保留左右两栏，但重新分配信息职责。

### 左侧：现在

左侧只负责呈现当前时间，不再承载宣传标题和说明文案。

展示顺序：

1. 彩虹短轨作为首屏视觉锚点。
2. `NOW · 现在` 小标题。
3. 实时 `时:分:秒`，每秒更新。
4. 公历日期与星期。
5. 农历日期。

小时和分钟作为主要视觉数字，秒数缩小并使用暖色强调，使时间持续走动但不过度抢眼。

### 右侧：重要时间

右侧由圆盘改为三行纵向时间轴，每行展示：

- 相差天数。
- `天后`、`今天` 或 `已过` 状态。
- 事件名称。
- 事件日期与星期。
- 置顶文字标记，仅用于置顶行。

事件行是只读信息，不响应点击，不展开详情，也不提供编辑或快捷操作。

## 4. 数据选择规则

右侧最多展示三项，按以下规则生成：

1. 如果存在置顶事件，第一行固定展示该事件；置顶事件可以是未来、今天或已过去的事件。
2. 从未被选中的未来事件中，按距离当前日期由近到远补足后两行。
3. 如果没有置顶事件，直接展示最近的三个未来事件。
4. 使用事件 `id` 去重，置顶事件不会再次出现在后续候选项中。
5. 候选不足三项时只展示现有项目，不制造占位数据。
6. 完全没有可展示事件时，右侧显示简短空状态“还没有重要日子”。

未来候选项覆盖自定义事件、周期事件与节假日，继续复用现有 `EventStore` 合并后的卡片数据和 `TimeCalc` 日期计算结果，不新增后端请求。

天数文案规则：

- 未来：`N 天后`。
- 今天：`就是今天`。
- 已过去：`已过 N 天`。

## 5. 视觉设计

新区域必须自然融入页面其余未修改部分，因此沿用 `public/css/fluffy.css` 现有设计变量：

- 奶油底色 `--cream`、`--cream-2`。
- 玻璃纸面 `--paper` 与现有毛玻璃渐变。
- 文字色 `--ink`、`--muted`。
- 珊瑚、奶油黄、薄荷绿、天空蓝和梅紫强调色。
- 统一 8px 圆角 `--radius`。
- 现有外阴影、内阴影和玻璃描边。

桌面端建议采用约 `0.78fr / 1.22fr` 的左右比例，让右侧三行时间轴获得更稳定的横向空间。

置顶行使用珊瑚到奶油黄的轻量暖色底纹，并同时显示“置顶”文字，不能只依赖颜色表达状态。其余两行使用薄荷绿、天空蓝短色条区分层级，不为每项增加独立主题卡片。

移除现有右侧圆盘、刻度线和漂浮动画。保留背景柔光与低强度玻璃高光，但首屏信息本身不使用循环位移动画。

## 6. 响应式布局

- 大于 980px：左右双栏，左侧“现在”，右侧“重要时间”。
- 不大于 980px：上下单栏，顺序固定为“现在 → 重要时间”。
- 不大于 680px：缩减面板内边距与字号，事件行保持单行名称、省略过长文本，不出现横向滚动。
- 不大于 420px：秒数、日期和事件日期进一步收敛，但天数、事件名称与置顶状态必须保持可见。

移动端不折叠、不轮播三条重要时间，避免隐藏核心信息。

## 7. 交互与数据流

数据流保持单向：

1. `home.js` 从 `EventStore` 取得合并后的卡片集合。
2. `CardRender` 根据置顶和未来日期规则选出最多三项。
3. `CardRender` 同时渲染当前时间面板与重要时间列表。
4. 现有每秒定时器更新时钟；跨越零点时重新计算日期、农历和事件天数。

首屏不产生写操作，因此不触发 `UIAlert.runExclusive()`，也不改变卡片置顶或排序状态。

## 8. 可访问性

- 左右区域使用有含义的 `section` 与标题关联。
- 重要时间使用有序列表语义，但不使用按钮或链接语义。
- 实时时钟不应通过 `aria-live` 每秒朗读；为辅助技术提供稳定的完整时间文本。
- 置顶状态同时使用文字和颜色。
- 所有文字与实际玻璃背景保持足够对比度。
- 继续尊重 `prefers-reduced-motion`，不引入必须播放的动画。

## 9. 异常与边界情况

- 置顶项已过去：保留第一行，显示“已过 N 天”。
- 置顶项也是最近未来项：通过 `id` 去重，后续只补其他事件。
- 事件不足三项：按实际数量渲染，容器不生成空行。
- 没有置顶项：展示最近三个未来事件。
- 没有任何未来项且没有置顶项：展示空状态。
- 事件名称过长：单行省略，并保留完整可访问名称。
- 时间跨越零点：日期、星期、农历和三条事件同步刷新。
- 数据加载失败：沿用现有首页错误处理，不在首屏新增独立错误弹窗。

## 10. 验证标准

实现完成后至少验证：

- 有未来置顶、过去置顶、无置顶三种数据状态。
- 置顶项不会在最近事件中重复。
- 零个、一个、两个、三个及三个以上候选项的渲染。
- 自定义事件、周期事件、节假日和已过事件的文案正确。
- 时钟每秒更新，跨日后所有日期相关信息刷新。
- 1240px、980px、680px、420px 和 320px 宽度下无重叠、裁切或横向滚动。
- 键盘与屏幕阅读器不会把只读事件行误识别为可操作控件。
- 修改过的 JavaScript 文件通过 `node --check`。
