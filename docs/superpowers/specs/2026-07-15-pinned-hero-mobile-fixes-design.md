# 置顶首屏与移动端适配修复设计

- 日期：2026-07-15
- 状态：已确认
- 范围：首屏置顶选择与文案、手机端密码页/浮动标题栏/卡片操作区

## 1. 背景与目标

当前置顶状态能正常写入 D1，`EventStore.getSortedCards()` 也会把置顶卡排在列表首位；但首屏“重要时间”在识别置顶项之前过滤了所有目标时间已过的卡片，导致“已过天数”和已到期倒计时无法显示在首屏第一行。

同时，现有 `max-width: 680px` 样式仍存在以下手机端问题：

- 密码页依次显示介绍区与密码卡，而不是只显示密码卡。
- 滚动后出现的浮动标题栏被改为纵向布局，占用过多高度。
- 手机端隐藏了真实“登出”文字，只依赖可能缺字的电源符号伪元素。
- 卡片操作区依赖 `:hover` / `:focus-within`，触屏设备默认不可见。

本次目标是在不改变桌面端、不修改后端与持久化语义的前提下修复上述问题。

## 2. 根因

### 2.1 置顶首屏失效

`public/js/card-render.js` 的 `getHeroMoments(cards)` 当前先通过 `TimeCalc.diff()` 判断目标是否已过，并立即排除 `isPast` 项，之后才从剩余候选中寻找 `pinned`。因此：

- 未来倒计时置顶正常。
- `elapsed` 事件通常会在置顶判断前被排除。
- 已到期倒计时以及当天目标时间已过的倒计时也会被排除。
- `EventStore`、D1 持久化和下方列表排序均正常，不是故障来源。

如果仅取消过滤，现有 `formatMomentCountdown()` 又会把过去项错误显示为“0 天后”，因此选择规则和方向文案必须一起修复。

### 2.2 手机布局问题

问题均来自 `public/css/fluffy.css` 的 `max-width: 680px` 覆盖：

- 密码页只改单列，没有隐藏 `.password-intro`。
- `.floating-header` 被设为纵向，`.header-actions` 又被撑到 `width: 100%`。
- `.logout-label` 被隐藏，按钮只剩 `::before` 的电源符号。
- `.card-actions` 的桌面默认值为透明且不可点击，手机断点没有覆盖。

## 3. 方案比较

### 方案 A：局部修复首屏选择器与手机断点（采用）

在 `card-render.js` 内调整首屏候选选择与过去文案；在现有 `max-width: 680px` 内增加精确覆盖；密码输入框补 `autofocus`。影响面最小，能保持 EventStore、D1、桌面布局和既有交互不变。

### 方案 B：修改 EventStore 的全局排序/过滤语义

让所有消费者依赖新的统一排序规则。该方案会把首屏展示策略耦合进数据层，并可能影响分类、拖拽和持久化顺序，不符合本次局部修复范围。

### 方案 C：重构首屏为“过去/未来”分区

可以完整表达时间方向，但需要新增信息架构、DOM 和响应式设计，超出用户提出的修复目标。

## 4. 详细设计

### 4.1 首屏置顶选择

`getHeroMoments(cards)` 使用以下顺序：

1. 先从所有可渲染卡片中找到唯一的 `pinned === true` 项，不按时间方向排除。
2. 将所有目标未到的非置顶项按目标日期升序排列。
3. 若存在置顶项，固定作为第一行，再用未来非置顶项补足，总数最多三行。
4. 若不存在置顶项，保持现状，仅展示最近三个未来事件。
5. 未置顶的过去事件继续排除，不占用“重要时间”的其余位置。
6. 解析日期失败的卡片不进入首屏候选；若该卡片置顶但无法解析，也不渲染错误行。

该规则保留全局单一置顶约束，不改变下方列表和拖拽排序。

### 4.2 时间方向文案

首屏时间文案保持对称：

- 未来：`N 天后`、`N 小时后`、`N 分钟后`、`即将`。
- 过去：`N 天前`、`N 小时前`、`N 分钟前`、`刚刚`。

过去文案只会用于置顶的过去项，因为未置顶过去项仍不进入首屏。

### 4.3 手机密码页

仅在 `max-width: 680px`：

- 隐藏 `.password-intro`。
- 让 `.password-artboard` 居中展示 `.password-card`，收紧垂直留白。
- 给 `#password-input` 增加 `autofocus`，保留 `password-init.js` 现有 `focus()` 作为后备。

移动浏览器，尤其 iOS Safari，可能禁止页面在没有用户手势时强制弹出软键盘。本实现提供标准允许范围内的自动聚焦与自动唤起，不加入循环 focus、模拟点击等不稳定技巧。

### 4.4 手机浮动标题栏与登出按钮

仅在 `max-width: 680px`：

- `.floating-header` 恢复横向排列并垂直居中。
- 隐藏标题块中的英文 eyebrow，保留“时光倒计时”。
- 标题不换行，字号和间距收紧。
- `.header-actions` 改为自动宽度、不换行并靠右。
- 恢复 `.logout-label` 的真实“登出”文字，移除电源符号伪元素，适当缩小按钮横向内边距。

`.list-toolbar` 与 `.modal-head` 保持原有手机布局，不跟随标题栏改动。

### 4.5 手机卡片操作区

仅在 `max-width: 680px` 将 `.card-actions` 设为默认可见、无位移并允许点击。桌面端继续使用 hover/focus 显示。

节假日仍遵守只读约定：只显示置顶按钮，不新增编辑或删除按钮。

## 5. 文件范围

- 修改 `public/js/card-render.js`：首屏选择规则与过去时间文案。
- 修改 `public/password.html`：密码输入框标准自动聚焦属性。
- 修改 `public/css/fluffy.css`：只增加或改写手机端断点覆盖。
- 新增 `test/card-render.test.js`：首屏置顶和方向文案回归测试。
- 新增 `test/mobile-ui-contract.test.js`：移动端 HTML/CSS 契约测试。

不修改：

- `src/worker.js`、D1 schema 和 API。
- `public/js/store.js` 的单一置顶、排序和持久化逻辑。
- 桌面端基础 CSS。
- 节假日只读规则。

## 6. 测试策略

仓库不引入新依赖，使用 Node 内置 `node:test` 与 `vm` 加载现有 IIFE 脚本。

测试驱动顺序：

1. 先写失败测试，证明过去的置顶 `elapsed` 和已到期 `countdown` 当前无法进入首屏第一行。
2. 先写失败测试，证明过去项当前错误显示为“0 天后”。
3. 先写失败契约测试，覆盖手机端密码介绍区、标题栏方向、登出文字、操作区可见性与 `autofocus`。
4. 实施最小修复，使测试转绿。
5. 运行所有新增测试与相关 JS 语法校验。

自动化断言包括：

- 过去置顶项固定为首屏第一行且 `isPinned === true`。
- 未来置顶项的现有行为不回归。
- 未置顶过去项仍不进入首屏。
- 过去文案正确覆盖天、小时、分钟和刚刚。
- 手机断点存在所需覆盖，密码输入框包含 `autofocus`。

浏览器验证矩阵：

- 390×844、420×844、680×900：密码页只显示密码卡；输入框获得焦点；浮动标题栏单行；登出文字可见；卡片按钮默认可见且可操作；无横向溢出。
- 1240px：密码页双区、标题栏和卡片 hover 行为与当前桌面端一致。

## 7. 完成标准

- 置顶的已过天数、已到期倒计时和未来倒计时均显示在首屏第一行。
- 过去与未来的方向文案正确，首屏最多三行且不重复。
- 手机密码页只显示密码卡，并在浏览器允许时自动唤起键盘。
- 手机浮动标题栏为单行且高度明显收紧，真实“登出”文字可见。
- 手机卡片操作按钮默认可见；桌面端行为不变。
- 新增测试全部通过，相关 JS 通过 `node --check`，浏览器断点矩阵无布局回归。
